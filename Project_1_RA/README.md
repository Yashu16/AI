 ## Explanations
 #### Search tool does:
 LLM requires tools to be defined in a specific format. This is an example of how to define a tool for web search. 
 The tool is defined with a function declaration that specifies the name, description, and parameters of the tool. 
 The LLM can then call this tool when it needs to perform a web search. 
 The response from the tool can be fed back into the conversation history for the LLM to use in generating its next response.
 So, function declaration -> name, desc, params(type, properties(query, type, desc), required(query))
 Parameters are filled by LLM when the user asks.

 #### Candidate[0] means the best answer among the candidates generated by Gemini. Gemini generates multiple candidates for each response, and we are taking the first one (index 0) as the best one to display to the user.


 And why do we have two while loops? Outer loop is for conversation and inner loop is for tool calling, and search-retreive and call itself again. Say we are searching: "compare DC vs Japan cherry blossoms" - Search for DC cherry blossoms -> loop back, search for Japan Cherry blossoms -> loop back and now compares them, calls itself again to get the final answer. 

 This is called ReAct (Reasoning + Acting) where the agent can reason about what tool to use and then act by calling the tool, and then use the results to reason further and act again until it reaches a final answer.

 #### Day_4: 
 Instead of letting gemini return raw final answer, we can let it return a JSON with summary and key facts, and then we can parse that JSON and display it in a more structured way to the user. This is more robust than relying on the model to format its response in a specific way, and allows us to handle the response more effectively.

 So far the agent talked back in plain conversational text. That's fine for a chatbot, but if we ever want to:

- Display results in a UI
- Save research to a file
- Feed output into another system

 we need structured output. Instead of Gemini rambling in paragraphs, you tell it to always return clean JSON in a shape you define.
